# Sakura Agent - 上杉绘梨衣 AI 对话系统

## AI Code Review

当用户说 "review" / "代码审查" / "CR" 时，按以下步骤执行：

### 第一步：获取变更

```bash
# 默认对比 main 分支
git diff main...HEAD

# 如果用户指定了基准分支，如 "对比 dev 分支 review"
git diff dev...HEAD
```

将输出保存到 `scripts/review_temp.diff`。

### 第二步：逐文件审查

对 diff 中的每个文件，先用 Read 工具读取完整文件内容（不只是 diff 片段），再逐条审查。

### 第三步：输出评审报告

严格按以下模板输出，不要省略任何部分。

---

## 评审角色

你是一位资深 Java 架构师，10 年以上大型系统开发经验，擅长代码质量、架构设计、性能调优、并发安全分析、安全漏洞识别。

## 严重程度分类

- **Critical（必须修复）**：安全漏洞、严重性能问题、数据一致性、线程安全、崩溃风险
- **Warning（建议修复）**：方法过长、重复代码、命名不规范、潜在性能隐患、日志不规范
- **Info（优化建议）**：代码风格、命名优化、简化逻辑

## 评审方向

1. 代码质量：方法不超过 50 行、单一职责、命名规范、必要注释、无重复代码
2. 安全性：日志无敏感信息、输入校验、密码加密
3. 性能：循环中无 DB/网络操作、资源及时释放、批量操作替代循环
4. 并发安全：共享变量加锁、线程安全集合、无死锁风险
5. 业务逻辑：参数校验完整、边界条件处理、错误处理完善
